  Telegram 筛号工具 v3.0 - 使用说明
═══════════════════════════════════════════════════════

🆕 v3.0 新功能：
  • 并发工作号数量可调（设置页 > 并发工作号数量）
  • 备用号自动替补：工作号出问题时，自动从待命账号里拉一个顶上
  • 队列并发筛号：每个账号独立拉号处理，筛选更快更稳

📦 首次使用步骤：

1. 获取 Telegram API 凭证
   - 访问 https://my.telegram.org
   - 登录你的 Telegram 账号
   - 点击 "API development tools"
   - 创建应用，获取 api_id 和 api_hash

2. 启动程序
   - Windows: 双击 TelegramFilter.exe
   - macOS: 双击 TelegramFilter.app

   首次运行 macOS 版本可能提示"无法打开"：
   - 打开"系统设置" > "隐私与安全性"
   - 找到被阻止的应用，点击"仍要打开"

3. 添加账号并登录
   - 点击"账号管理"标签页
   - 点击"添加账号"
   - 填写信息：
     * 账号名称（如 account1）
     * API ID（从第1步获取）
     * API Hash（从第1步获取）
     * 手机号（带国家码，如 +8613800138000）
     * 代理（可选，格式：socks5://127.0.0.1:1080）
   - 点击"保存"
   - 在账号列表中，点击账号右键选择"登录"
   - 输入 Telegram 发送的验证码
   - 如果有两步验证，输入密码
   - 登录成功后状态显示为"已登录"

4. 开始筛选
   - 点击"筛选"标签页
   - 输入或导入手机号（每行一个）
   - 选择目标国家（美国/中国）
   - 设置探针验证（可选，防止误判）
   - 点击"开始筛选"
   - 筛选完成后可导出结果

📁 文件说明：

TelegramFilter.exe / TelegramFilter.app - 主程序（包含登录和筛选功能）
config.json - 账号配置文件（程序自动生成）
session_*.session - 登录会话文件（登录后自动生成）
registered_*.txt - 已注册号码导出文件
unregistered_*.txt - 未注册号码导出文件
filter_progress.json - 筛选进度文件

⚠️ 注意事项：

1. API 凭证安全
   - 不要分享你的 api_id 和 api_hash
   - 不要上传 config.json 到公开位置

2. Session 文件
   - session 文件包含登录信息，请妥善保管
   - 如果更换电脑，可以复制 session 文件避免重新登录

3. 速率限制
   - Telegram 有速率限制，建议使用多个账号
   - 如果触发限制，程序会自动切换账号

4. 代理使用
   - 如果网络受限，建议配置 SOCKS5 代理
   - 代理配置错误会导致连接失败

5. 探针验证
   - 建议开启探针验证，防止批量误判
   - 探针号码应该是已知注册的号码

6. 并发工作号数量（v3.0 新增）
   - 位置：⚙️ 设置页 > "并发工作号数量"
   - 作用：决定同时在线跑号的账号数；其余账号作为"备用号"待命
   - 推荐：账号够用时配 (总数 - 2)，留 2 个做应急替补
         例：5 个号 → 并发 3，剩 2 个作备用
   - 全开（等于账号总数）吞吐最高，但出事时没有备用顶上
   - 修改后记得点"💾 保存设置"，下一轮筛选生效

7. 备用号自动替补（v3.0 新增）
   - 当某个工作号探针连续失败被暂停，系统会：
     a. 从备用号里自动挑一个登录可用的升级为工作号
     b. 新工作号立即接入队列继续跑号
     c. 日志会显示"🔁 备用号 X 已接替 Y"
   - 所有账号（含备用号）都暂停后，才会紧急停止筛选
   - 备用号必须已登录（有 session 文件）才会被启用

❓ 常见问题：

Q: 提示"手机号无效"？
A: 检查手机号格式，必须包含国家码（如 +86）

Q: 提示"连接失败"？
A: 检查网络连接，或配置代理

Q: 提示"代理认证失败"？
A: 检查代理用户名和密码是否正确

Q: 如何添加多个账号？
A: 在 config.json 的 accounts 数组中添加多个账号配置

Q: 如何更换电脑使用？
A: 复制以下文件到新电脑：
   - TelegramFilter.exe / TelegramFilter.app
   - config.json
   - session_*.session（可选，避免重新登录）

📧 技术支持：

如有问题，请检查：
1. config.json 格式是否正确（JSON 格式）
2. API 凭证是否有效
3. 网络连接是否正常
4. 代理配置是否正确

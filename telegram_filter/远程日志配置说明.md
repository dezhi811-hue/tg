# 远程日志配置说明

远程日志功能可以将程序的崩溃和异常信息自动发送到你的 Telegram，方便诊断问题。

## 配置步骤

### 1. 创建 Telegram Bot

1. 在 Telegram 中搜索 `@BotFather`
2. 发送 `/newbot` 命令
3. 按提示设置 Bot 名称和用户名
4. 获得 Bot Token，格式类似：`1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`

### 2. 获取 Chat ID

方法一：使用 @userinfobot
1. 在 Telegram 中搜索 `@userinfobot`
2. 点击 Start 或发送任意消息
3. Bot 会返回你的 Chat ID（纯数字）

方法二：使用 @getidsbot
1. 在 Telegram 中搜索 `@getidsbot`
2. 点击 Start
3. 获取你的 Chat ID

### 3. 配置 config.json

在 `config.json` 中添加以下配置：

```json
{
  "accounts": [...],
  "rate_limit": {...},
  "remote_log": {
    "enabled": true,
    "bot_token": "你的Bot_Token",
    "chat_id": "你的Chat_ID"
  }
}
```

**示例：**
```json
{
  "remote_log": {
    "enabled": true,
    "bot_token": "1234567890:ABCdefGHIjklMNOpqrsTUVwxyz",
    "chat_id": "123456789"
  }
}
```

### 4. 启动 Bot

1. 在 Telegram 中找到你创建的 Bot
2. 点击 Start 或发送 `/start`
3. 这样 Bot 才能给你发送消息

## 日志级别

远程日志会发送以下级别的消息：

- ℹ️ **INFO**: 程序启动等一般信息
- ⚠️ **WARNING**: 账号检测异常等警告
- ❌ **ERROR**: 筛选过程中的错误
- 🔥 **CRITICAL**: 程序崩溃等严重错误

## 隐私说明

- 日志只发送到你自己的 Telegram Bot
- 不会发送到任何第三方服务器
- 日志内容包含：错误信息、堆栈跟踪、系统信息
- 不会包含：手机号、API密钥等敏感信息

## 关闭远程日志

如果不需要远程日志，设置 `enabled: false` 即可：

```json
{
  "remote_log": {
    "enabled": false,
    "bot_token": "",
    "chat_id": ""
  }
}
```

或者直接删除 `remote_log` 配置项。

## 测试

配置完成后，启动程序，你应该会收到一条消息：

```
ℹ️ INFO

消息: 筛号工具已启动
时间: 2026-04-16 02:00:00
平台: Windows 10.0.19041
```

如果收到这条消息，说明配置成功！

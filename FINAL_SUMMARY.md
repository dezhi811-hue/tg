## 🎯 核心功能

### ✅ 已实现功能

1. **多账号轮换系统**
   - 支持3-10个Telegram账号同时管理
   - 智能账号切换，避免单账号过载
   - 每个账号独立统计和监控
   - 自动处理账号封禁和恢复

2. **智能防封策略**
   - 随机延迟：每次请求间隔3-8秒（可配置）
   - 请求限制：单账号连续请求不超过30次
   - FloodWait处理：触发速率限制自动切换账号
   - 错误重试：失败自动重试最多3次
   - 冷却机制：错误后增加延迟时间
   - 自动暂停：请求过于频繁时自动休息

3. **美国号码支持**
   - 自动识别和格式化多种美国号码格式
   - 支持格式：+1xxx, xxx, (xxx) xxx-xxxx等
   - 自动添加国家码+1
   - 批量格式化和验证

4. **用户活跃度检测**
   - 🟢 online - 当前在线
   - 🟡 recently - 最近在线（几分钟到几小时）
   - 🟠 within_week - 一周内在线
   - 🔴 within_month - 一个月内在线
   - ⚫ offline - 离线（显示具体时间）
   - ⚪ long_ago - 很久未上线

5. **图形界面（GUI）**
   - 可视化账号管理（添加/编辑/删除）
   - 直接输入或导入号码文件
   - 实时查看筛选进度和日志
   - 统计信息可视化
   - 一键导出结果（CSV/JSON）

6. **命令行界面（CLI）**
   - 适合自动化脚本
   - 服务器环境运行
   - 批量任务处理

## 🚀 快速开始

### GUI版本（推荐新手）

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动GUI
python gui.py

# 3. 在GUI中：
#    - 切换到"账号管理"，添加3-5个账号
#    - 切换到"筛选"，输入号码
#    - 点击"开始筛选"
```

### CLI版本（推荐高级用户）

```bash
# 1. 配置账号
cp config.example.json config.json
# 编辑config.json，添加账号信息

# 2. 准备号码文件
echo "+12025551234" > us_phones.txt

# 3. 运行筛选
python main_multi.py --file us_phones.txt --country US --output result.csv
```

2. 登录你的Telegram账号

4. 创建应用，获取 `api_id` 和 `api_hash`
5. 为每个账号重复此过程

## 🛡️ 防封策略

### 多重防护机制

1. **多账号轮换** - 自动在多个账号间切换
2. **智能延迟** - 每次请求随机延迟3-8秒
3. **请求限制** - 单账号连续请求不超过30次
4. **错误处理** - 遇到FloodWait自动切换账号
5. **自动重试** - 失败自动重试最多3次
6. **冷却机制** - 错误后增加延迟时间

### 使用建议

✅ **推荐做法：**
- 使用3-5个真实手机号注册的老账号
- 延迟时间设置3-8秒
- 分批次执行，每批100-500个号码
- 每批次后休息几小时
- 避免24小时连续运行

❌ **避免做法：**
- 使用虚拟号码注册的新账号
- 延迟时间设置过短（<2秒）
- 一次性筛选大量号码（>1000）
- 长时间连续运行
- 单账号高频请求
